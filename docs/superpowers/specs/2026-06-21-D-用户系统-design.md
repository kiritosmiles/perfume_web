## 概述

在 MVP Phase 1 游客模式基础上，建立用户身份系统，支持 Free 用户注册/登录、JWT 认证、日配额管理、Guest → Free 数据迁移。

| ID | 功能 | 性质 | 预估 |
|----|------|------|------|
| D1 | 认证后端 (register/login/refresh) | 后端 | 1.5h |
| D2 | 前端 Auth UI (LoginPage/RegisterPage) | 前端 | 1h |
| D3 | ProtectedRoute + AuthChatPage | 前端 | 1h |
| D4 | 用户配额 + Guest 数据迁移 | 后端 | 1h |
| D5 | 推荐组件提取 (RecommendationFlow) | 前端 | 1h |

| 决策点 | 选择 |

## 后端设计

### 数据库

**users table (Alembic 003):**
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**refresh_tokens table:**
```sql
CREATE TABLE refresh_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash VARCHAR(255) UNIQUE NOT NULL,
    expires_at TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_refresh_tokens_user ON refresh_tokens (user_id);
```

**user_quota table:**
```sql
CREATE TABLE user_quota (
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    quota_type VARCHAR(32) NOT NULL CHECK (quota_type IN ('sessions', 'generations', 'deep')),
    used INTEGER NOT NULL DEFAULT 0,
    reset_at DATE NOT NULL DEFAULT CURRENT_DATE,
    PRIMARY KEY (user_id, quota_type, reset_at)
);
```

### API 端点 (`backend/app/api/v1/auth.py` — 新建)

**POST /api/v1/auth/register**
```
Request:  { email: str, password: str, browser_id?: str }
Response: { user: {id, email}, access_token: str, refresh_token: str }
Logic:
  1. validate email format + password >= 8 chars
  2. check email uniqueness → 409 if taken
  3. hash password, INSERT users
  4. if browser_id provided → migrate temp_conversations to user_id
  5. generate token pair, INSERT refresh_tokens (hash)
  6. return user + tokens
```

**POST /api/v1/auth/login**
```
Request:  { email: str, password: str }
Response: { user: {id, email}, access_token: str, refresh_token: str }
Logic:
  1. lookup user by email → 401 if not found
  2. verify password → 401 if mismatch
  3. generate token pair, INSERT refresh_tokens
  4. return user + tokens
```

**POST /api/v1/auth/refresh**
```
Request:  { refresh_token: str }
Response: { access_token: str, refresh_token: str }
Logic:
  1. decode refresh_token → 401 if invalid/expired
  2. lookup token_hash in refresh_tokens → 401 if not found
  3. DELETE old refresh token (rotation)
  4. generate new token pair, INSERT new refresh token
  5. return tokens
```

**GET /api/v1/auth/me**
```
Headers: Authorization: Bearer <access_token>
Response: { id: str, email: str, created_at: str }
Logic:
  1. decode access_token → 401 if invalid
  2. lookup user → 200 with user info
```

## 安全

- 密码: bcrypt 哈希（cost=12），绝不存储明文
- JWT: HS256 对称签名，生产环境 JWT_SECRET ≥ 32 字节随机字符串
- Refresh Token 轮换: 每次 refresh 删除旧 token，防止重放
- Refresh Token 存储: PG 存储 hash（SHA-256），不存原文
- 密码最短 8 字符，无其他复杂度要求（MVP 不做过度约束）
- 同一邮箱重复注册 → 409 Conflict

## 回退策略

| 场景 | 行为 |
|------|------|
| JWT_SECRET 未配置 | 使用 `perfume-dev-secret`（仅开发），生产打印 warning |
| Refresh Token 被盗用 | Token 轮换 + 单次使用 → 旧 token 失效 |
| 游客数据迁移失败 | 注册仍成功，游客数据保留 30 天（不丢失） |
| bcrypt 不可用 | passlib 回退到内置哈希（开发降级） |

- 邮件验证
- 密码重置
- OAuth 第三方登录
