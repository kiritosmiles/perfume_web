## C2: LLM API Key 前端配置

### 目标

用户在前端设置页填入自己的 OpenAI 兼容 API Key（如 DeepSeek），后端请求时优先使用用户 Key，实现"自带 Key"模式。

### 数据流

```
[SettingsPage] ──POST /api/v1/config/llm-key──→ Redis
                   {browser_id, api_key}          SET llm_key:{bid} <key> EX 86400

[SSE 请求] ──读取 browser_id──→ get_llm_key(bid)
                                  ├── Redis 命中 → 覆盖 settings.LLM_API_KEY
                                  └── Redis 缺失 → fallback 系统 Key（或模板回退）
```

### 后端改动

| 文件 | 改动 |
|------|------|
| `backend/app/api/v1/config.py` (新建) | `POST /config/llm-key` — 接收 `{browser_id, api_key, base_url?}`，验证后存入 Redis。`GET /config/llm-key/status?browser_id=xxx` — 检查是否已配置 |
| `backend/app/api/v1/router.py` | 注册 `/config` 路由 |
| `backend/app/core/redis.py` | 新增 `store_llm_key(bid, key, base_url?)` → `SET llm_key:{bid} <json> EX 86400`；`get_llm_key(bid) → dict\|None` |
| `backend/app/services/llm.py` | 调用前 `get_llm_key(browser_id)` → 有则覆盖实例属性 |
| `backend/app/services/llm_emotion.py` | 同上 |

### 前端改动

| 文件 | 改动 |
|------|------|
| `packages/frontend/src/routes/SettingsPage.tsx` (新建) | 表单：API Key（密码输入框）+ Base URL（选填，默认 DeepSeek）+ 保存按钮 + 配置状态指示。存储标记 `llm_key_configured` 到 localStorage |
| `packages/frontend/src/App.tsx` | 添加 `<Route path="/settings" element={<SettingsPage />} />` |
| `packages/frontend/src/lib/apiClient.ts` | 新增 `saveLLMKey(browserId, apiKey, baseUrl?)` → POST |

### 安全

- Key 仅通过 POST body 传输一次，不在 URL 出现
- Redis 127.0.0.1 绑定，外部不可达
- 前端 localStorage 仅存布尔标记，不存 Key 原文
- TTL 24h 后自动过期，用户需重新输入（或后续 B4 用户系统后可持久化）

### 回退策略

| 场景 | 行为 |
|------|------|
| 用户未配置 Key | 使用系统 `LLM_API_KEY` 或模板回退 |
| 用户 Key 过期/失效 | LLM 调用返回 401 → 降级模板回退 + 前端 toast "API Key 可能已失效" |
| Redis 不可用 | `get_llm_key` 返回 None → 降级系统 Key |

## C3: 动态笔记 PNG 导出

### 目标

生成完成后，用户可将推荐结果保存为精美 PNG 笔记卡片。

### 组件结构

```
NoteCard (新建)
├── NoteHeader          — 日期 "2026.06.21"、标题 "Perfume Note"
├── EmotionBadge        — primary_emotion 中文名 + confidence 百分比
├── FragranceEntries    — 3 款推荐
│   ├── #1 名称 | 品牌 | 香调组合 tags | match_score 进度条
│   ├── #2 ...
│   └── #3 ...
├── NoteFooter          — 品牌 slogan "Emotion × Fragrance" + 时间戳 + scene_tag
└── 玻璃拟态背景        — 与登入页风格一致的渐变 + 毛玻璃层
```

### 导出管线

```
[点击 "Save as Note"]
  → useRef 指向 NoteCard DOM 节点 (display: none)
  → import { toPng } from "html-to-image"
  → toPng(noteRef, { backgroundColor: "#f5f0eb", pixelRatio: 2 })
  → blob → URL.createObjectURL → <a download="perfume-note-2026-06-21.png">
  → 清理 URL
```

### 文件清单

| 文件 | 说明 |
|------|------|
| `packages/frontend/src/components/notes/NoteCard.tsx` (新建) | 笔记卡片（拿 generationStore cards + sessionStore emotion 渲染） |
| `packages/frontend/src/components/notes/ExportButton.tsx` (新建) | 按钮 + html-to-image 调用 + loading/error/success 三态 |
| `packages/frontend/src/routes/GuestChatPage.tsx` | `phase === "complete"` 时渲染 `ExportButton + <div ref={noteRef} className="absolute -left-[9999px]"><NoteCard /></div>` |

### 新增依赖

`html-to-image` (~5KB gzipped)

### 导出内容（B 方案：含情绪分析）

- 日期
- primary_emotion + confidence
- 3 款推荐：名称、品牌、香调组合、match_score、copy_text（截断前 120 字）
- scene_tag（如有）
- 水印 "perfume-ai.vercel.app"

## C4: 分享链接

### 目标

用户生成一次推荐后，点击"分享"生成短链接，其他人访问可看到只读推荐卡片。

### 数据流

```
[点击 Share] → POST /api/v1/share
                 { recommendations: [...], emotion: {...}, scene_tag, generation_id }
                 → PG INSERT share_links (id, payload, created_at, expires_at)
                 → 返回 { share_url: "/s/<id>", share_id }

[访问 /s/:id] → SharePage 挂载
                 → GET /api/v1/share/:id
                 → PG SELECT → 返回 payload JSON
                 → 渲染只读 ShareCard（复用 NoteCard 样式但不可编辑）
```

### 数据库

```sql
CREATE TABLE share_links (
    id CHAR(8) PRIMARY KEY,
    payload JSONB NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    expires_at TIMESTAMPTZ NOT NULL DEFAULT (now() + INTERVAL '7 days')
);
CREATE INDEX idx_share_expires ON share_links (expires_at);
```

ID 用 `nanoid` (8 位字母数字，36^8 ≈ 2.8 万亿组合)。

### 后端改动

| 文件 | 改动 |
|------|------|
| `backend/app/api/v1/share.py` (新建) | `POST /share` — 生成短链接，payload 通过浏览器 JS `structuredClone` + JSON 构造。`GET /share/{id}` — 查询 + 返回 payload + 过期检查 |
| `backend/app/api/v1/router.py` | 注册 `/share` 路由 |
| `backend/alembic/versions/002_share_links.py` (新建) | 建 `share_links` 表 migration |

### 前端改动

| 文件 | 改动 |
|------|------|
| `packages/frontend/src/routes/SharePage.tsx` (新建) | 路由 `/s/:id`。挂载时 `GET /api/v1/share/:id` → Loading → 渲染只读 ShareCard 或 404 过期提示 |
| `packages/frontend/src/components/share/ShareCard.tsx` (新建) | 纯展示卡片 — 情绪标签 + 3 款推荐列表（复用 NoteCard 视觉样式） |
| `packages/frontend/src/App.tsx` | 添加 `<Route path="/s/:id" element={<SharePage />} />` |
| `packages/frontend/src/routes/GuestChatPage.tsx` | 完成阶段显示 Share 按钮 → 调用 POST `/share` → toast 弹出复制链接按钮 |

### 分享页面设计

- 深色渐变背景（与 LandingPage 一致）
- 居中玻璃拟态卡片
- 顶部：情绪标签 + "via Perfume AI"
- 中部：3 款推荐（排名、品牌、香调、分数、文案摘要）
- 底部：时间戳 + "体验你自己的 → 开始探索" CTA 按钮（链接首页）

### 过期处理

- 7 天后自动过期
- SharePage: "这个分享链接已过期，自己去体验一次吧 → 开始"
- 后端 GET 返回 410 Gone

GitHub Actions CI 新增 `e2e` job：
```yaml
e2e:
  runs-on: ubuntu-latest
  services:
    postgres:
      image: postgres:15-alpine
      env: { POSTGRES_USER: perfume, POSTGRES_PASSWORD: perfume_dev, POSTGRES_DB: perfume }
    redis:
      image: redis:7-alpine
      # ...
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: "3.11"
    - run: pip install poetry
    - run: cd backend && poetry install --no-interaction --no-root
    - name: Install Playwright

```
