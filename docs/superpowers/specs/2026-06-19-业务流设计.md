## 2. C 端全链路流程

### 2.1 完整旅程图（含错误/恢复路径）

现有需求文档附录 A 定义了 13 步 happy path。以下为每一步补充 ✅ 成功 / 🟡 降级 / 🔴 失败三态路径。

```
用户旅程主流程

  ┌─────────────────────────────────────────────────────────────────────┐
  │ ① 进入应用                                                           │
  │   ✅ 有 cookie → 识别身份 → 预加载画像 + Layer 2 热检索              │
  │   🟡 无 cookie → 展示游客入口 + 登录入口（详见 FR-1.7）               │
  │   🔴 服务不可用 → 静态降级页「稍后回来 🌿」（不含任何动态功能）       │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ② 情绪输入                                                            │
  │   ✅ 文字输入 / 情绪卡片二选一（详见 FR-2.1）                         │
  │   🟡 用户犹豫 > 30s → 展示引导气泡「说说今天的心情？或选一张卡片」    │
  │   🔴 输入层超时(§1.9 ①②) → 走 BERT 低置信结果 + 降级标记            │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ③ 意图检测                                                            │
  │   ✅ 判定 self_use / gift / explore（详见 FR-5.7）                    │
  │   🟡 置信度 < 0.85 → 反问确认（详见 FR-4.7）                         │
  │   🔴 判定失败 → 默认 self_use，标记 intent_fallback                   │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ④ 改写+过滤                                                           │
  │   ✅ 改写按需触发 / 敏感词过滤异步旁路（详见 FR-5.2）                  │
  │   🟡 敏感词命中 → 前端不展示用户原话，替换为「消息已发送 ✅」         │
  │   🔴 调用 B 超时 → 纯本地敏感词降级，标记 safety_degraded              │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑤ 情绪识别（详见 FR-2.2）                                             │
  │   ✅ BERT 高置信(≥0.7) → 直接使用，~50ms                              │
  │   🟡 BERT 低置信(<0.7) → LLM 兜底，~800ms                            │
  │   🟡 情绪确认微交互展示 → 用户可在 3s 内点「不对」修正（详见 FR-2.3）  │
  │   🔴 BERT + LLM 均失败 → 用场景标签推断默认情绪分布                    │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑥ 上下文加载（意图门控）                                               │
  │   ✅ Layer 1+2+3 + GraphRAG 按意图加载（详见 FR-4.10）                │
  │   🟡 Layer 2 热检索相似度 < 0.5 → 异步触发冷归档（下一轮注入）        │
  │   🟡 GraphRAG 2 跳 0 结果 → 扩展到 3 跳 → 仍 0 → LLM 纯知识回退      │
  │   🔴 所有层均超时 → 仅用 Layer 1 当前窗口 + 通用图谱路径              │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑦ Agent Gate: 信息完整性（详见 FR-5.11）                               │
  │   ✅ sufficient(95%) → 跳过，0ms 开销                                 │
  │   🟡 partial(3%) → 调 1 次工具补充，+200ms                            │
  │   🔴 insufficient(2%) → 输出追问，不进入生成                          │
  │   🔴 Gate 超时(500ms) → bypass，标记 gate_bypassed，继续生成          │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑧ 推理生成                                                             │
  │   ✅ 快速模式(88%) → 单次 LLM + Prompt Cache，骨架 ~1.3-1.5s          │
  │   ✅ 深度模式(12%) → Supervisor + Subagent，骨架 ~1.7s（详见 FR-3.12）│
  │   🟡 LLM 超时 → 重试 1 次 → 仍失败 → 降级（缓存/规则/图谱纯检索）     │
  │   🔴 全部降级失败 → 输出通用推荐卡片墙（Top 5 高频香调）               │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑨ SSE 流式输出（详见 FR-4.13, FR-5.4）                                │
  │   ✅ skeleton → details → copy 三阶段流式                             │
  │   🟡 骨架阶段断连 → localStorage 记录 generation_id + phase           │
  │       重连后从前端发送 resume → 后端从断点继续                        │
  │   🔴 generation_id 已过期(>5min) → 重新生成                           │
  │   🔴 生成中检测到 crisis → SSE 中断 + 前端切换危机模式                │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑩ 精炼对话（详见 FR-3.7）                                              │
  │   ✅ 第 1 次 → 规则引擎，< 500ms，9 维指令覆盖                        │
  │   🟡 第 2 次 → Agent Gate 语义判断(200ms)，是否追问                   │
  │   🟡 意图变化(精炼前) → 立即确认，不等第 2 次 Gate                     │
  │   🔴 第 3 次仍失败 → 升级深度模式 + 文案「换个思路 🌿」                │
  │   🔴 深度模式后仍不满意 → 收藏所有版本 + 建议转人工                    │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑪ 反馈收集                                                             │
  │   ✅ 隐式反馈异步记录（详见 FR-7.1）                                   │
  │   🟡 显式反馈（❤️/💤）→ 延后到下次打开时询问（详见 FR-7.2）            │
  │   🔴 埋点上报失败 → 本地缓冲，下次重试（最多 50 条）                   │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑫ 存储总结                                                             │
  │   ✅ Session 结束 → 异步生成摘要 → Layer 2 双写（详见 FR-4.5）         │
  │   🟡 微会话合并（< 2min）→ 原文保留，摘要合并到下一长会话（FR-4.6）    │
  │   🔴 摘要生成失败 → 重试 1 次 → 仍失败 → 保存原文摘要为会话标题       │
  ├─────────────────────────────────────────────────────────────────────┤
  │ ⑬ 长期留存                                                             │
  │   ✅ 主动回访通知（详见 FR-4.9 四种触发条件）                          │
  │   🟡 Web Notification 被拒 → 回退邮件渠道（opt-in）                    │
  │   🔴 邮件退信 → 标记 bounce，不重试，等待下次回访条件触发              │
  └─────────────────────────────────────────────────────────────────────┘
```

### 2.2 用户状态机

```
                      ┌──────────────┐
                      │    游客       │
                      │   (Guest)    │
                      └──┬───┬───┬──┘
                         │   │   │
             注册(FR-1.9)│   │   │ 30天未注册 → 数据物理删除
                         │   │   │
              ┌──────────┘   │   └──────────┐
              ▼              ▼              ▼
       ┌──────────┐   ┌──────────┐   ┌──────────┐
       │  新用户   │   │  游客离开  │   │  游客回归  │
       │ (New)    │   │  无转化    │   │  再对话    │
       └────┬─────┘   └──────────┘   └──────────┘
            │
            │ 第 4 次对话起
            ▼
     ┌──────────┐        > 7 天未访问        ┌──────────┐
     │  老用户   │ ────────────────────────→ │  不活跃   │
     │ (Regular) │ ←──────────────────────── │ (Dormant) │
     └────┬─────┘    回来继续对话             └────┬─────┘
          │                                       │
          │                              > 3 个月未访问
          │                                       │
          │         ┌──────────┐                  │
          │         │  回归用户  │ ←───────────────┘
          │         │ (Returning)│   轻量重新引导 ≤2题
          │         └────┬─────┘   选"生活大变化"→降权50%
          │              │         冷归档第2轮后才启动
          │   连续 3 次     │
          │   overstep      │
          ▼                ▼
    ┌──────────┐    ┌──────────┐
    │  建议转人工 │    │  回到     │
    │  或新Session│   │  Regular  │
    └──────────┘    └──────────┘
```

| 状态 | 定义 | 进入条件 | 退出条件 | 关联 FR |
|------|------|---------|---------|--------|
| `guest` | 游客 | 首次访问无 cookie | 注册 / 30 天过期 | FR-1.7 |
| `new` | 新用户 | 注册成功 | 第 4 次对话 | FR-1.3 |
| `regular` | 老用户 | 从 new 升级 / 从 dormant 恢复 | 连续 7 天未访问 | — |
| `dormant` | 不活跃 | 连续 7 天未访问 | 用户回来 / 回访通知触发 | FR-4.9 |
| `returning` | 回归 | 连续 3 个月未访问后回来 | 轻量引导完成后 → regular | §10.7 |

### 2.3 会话状态机

```
       ┌────────┐
       │  idle   │ ← 用户进入/对话完成/精炼完成
       └───┬────┘
           │
           │ 用户发送消息
           ▼
     ┌──────────┐
     │ loading  │ ← 加载画像 + 记忆检索 + GraphRAG
     │          │   展示骨架屏/打字动画
     └────┬─────┘
          │
     ┌────┴────┐
     │         │
     ▼         ▼
┌────────┐  ┌──────────────┐
│awaiting│  │generating_    │
│_answer │  │skeleton       │ ← SSE: phase=skeleton
│        │  │               │   展示香调组合卡片
│  追问   │  └──────┬───────┘
│  输出   │         │
└──┬─────┘    ┌────┴────┐
   │          │         │
   │          ▼         ▼
   │    ┌─────────┐ ┌─────────┐
   │    │generating│ │crisis_  │ ← crisis 检测 → SSE 中断
   │    │_details │ │paused   │   暂停推荐，展示关怀
   │    │         │ │         │
   │    │ SSE:    │ │ 本轮     │
   │    │ details │ │ session  │
   │    │ + copy  │ │ 禁止推荐 │
   │    └────┬────┘ └────┬────┘
   │         │           │
   │         ▼           │ 用户关闭/
   │    ┌────────┐       │ 新 session
   │    │refining│ ←用户说"太甜了"等│
   │    │        │       ▼
   │    └───┬────┘  ┌────────┐
   │        │       │  idle   │ (新一轮，crisis 状态已清除)
   │   精炼完成     └────────┘
   │   (第1/2次)
   │        │
   │   精炼3轮失败 → 升级深度模式 → generating_skeleton
   │        │
   │   深度后仍不满意 → 收藏 + 转人工
   │        │
   │        ▼
   │   ┌──────────┐
   │   │ completed │ ← 用户主动结束 / 空闲超时 30min
   │   └──────────┘
   │
   └─ 用户回答追问 → loading (重新进入 Gate)
```

| 状态 | 说明 | 用户可操作 | 超时行为 |
|------|------|-----------|---------|
| `idle` | 等待用户输入 | 输入文字、选卡片、切换意图、浏览历史 | 30min → `completed`（详见 FR-4.1） |
| `loading` | 加载上下文 | 不可操作（输入框禁用） | — |
| `awaiting_answer` | 等待用户回答 Gate 追问 | 文字回答 / 跳过「先推荐看看」 | 不回答不超时，用户可忽略继续发消息 |
| `generating_skeleton` | 骨架流式中 | 不可中断（正常场景） | 见需求文档 §1.9 |
| `generating_details` | 详情流式中 | 可切换标签页等待 | — |
| `refining` | 精炼调整中 | 可发新精炼指令 | — |
| `crisis_paused` | 危机暂停 | 自由对话，不可触发推荐 | 本 session 不解锁，新 session 恢复 |
| `completed` | 会话结束 | 查看历史、开始新对话 | — |

### 2.4 关键回流路径

#### 2.4.1 中断恢复（关闭浏览器后回来）

```
用户关闭/切走浏览器
    │
    ├── Session 仍活跃（< 30min，详见 FR-4.1）:
    │   前端重新打开 → 检测 localStorage 中 session_id
    │   → GET /session/{session_id}/status
    │   → {status: "active", last_message_at: "...", has_pending_generation: true/false}
    │   ├── 无未完成生成 → 加载最近 6 轮对话 → idle 状态
    │   └── 有未完成生成(generation_id 未过期，< 5min):
    │       → POST /generation/{generation_id}/resume {from_phase: "skeleton"}
    │       → 后端从对应 phase 重新推送 SSE
    │
    └── Session 已过期（> 30min）:
       前端清除 session_id → 展示"上次对话已结束 🌿"
       → 点击"开始新对话" → 新 session_id，继承历史上下文(Layer 1+2)
```

#### 2.4.2 通知回流（点击 Web Notification）

```
用户点击通知
    │
    ├── 通知类型: 情绪周记 / 季节推荐 / 卡片完成
    │
    ├── 浏览器已打开:
    │   → 前端接收 notificationclick 事件
    │   → 导航到对应页面:
    │      - 情绪周记 → /mood-journal/{week_id}
    │      - 卡片完成 → /my-cards → 高亮新卡片
    │      - 季节推荐 → /chat?scene={season}&mode=explore（新对话）
    │
    └── 浏览器未打开:
       → 打开新标签页 → 检查登录状态
       ├── 已登录 → 直接进入对应页面
       └── 游客 → 引导注册/登录（游客不接收通知，此为注册用户路径）
```

#### 2.4.3 分享回流（被分享者打开链接）

```
被分享者点击分享链接 / 扫描二维码
    │
    ├── 链接格式: /share/{share_id}
    │   内容: 推荐的香调卡片 + AI 解读文案（只读，不含用户情绪数据）
    │
    ├── 新用户:
    │   → 展示分享卡片（静态预览，含香调名 + 解读文案摘录）
    │   → CTA: 「发现你的专属香调 🌿」→ 进入游客模式
    │
    └── 已注册用户:
       → 展示分享卡片
       → CTA: 「试一次类似推荐」→ 新对话，以分享卡片的情绪+场景为初始输入
```

#### 2.4.4 历史浏览 → 恢复对话

```
用户浏览历史会话（详见 FR-4.11 按天聚合视图）
    │
    ├── 点击某个历史 session:
    │   → /history/{session_id} — 只读视图，展示完整对话 + 推荐结果
    │   → 「继续这个话题 🌿」按钮
    │       ├── 该 session 的配方 → 作为新 session 的初始参考（非精炼起点）
    │       └── 新 session_id，Layer 1 注入该历史 session 的摘要
    │
    └── 点击历史 session 中的某次推荐:
       → 进入配方详情卡片（只读）
       → 📋 「以这个为基础调整」→ 新 session，该配方作为初始推荐骨架
       → 📤 「分享」→ 生成分享链接
```

## 3. 跨模块数据流

### 3.1 模块间接口契约

#### 3.1.1 模块依赖拓扑

```
                        ┌─────────────┐
                        │  M5: 对话与安全 │
                        │  (入口编排)    │
                        └──┬──┬───┬───┘
                           │  │   │
              ┌────────────┘  │   └────────────┐
              ▼               ▼                ▼
```

### 3.2 事件总线设计

系统采用 **Redis Pub/Sub + 应用层 EventRouter** 的轻量事件总线。事件分为同步处理（阻塞当前请求）和异步处理（后台消费）。

#### 3.2.1 命名规范

```
事件命名规范: {domain}.{action}.{status}
示例: emotion.classified.success
      crisis.detected.high
      session.completed
```

#### 3.2.2 事件目录

| 事件 | 发布者 | 消费者 | 处理方式 | 延迟要求 |
|------|--------|--------|---------|---------|
| `emotion.classified` | M2 | M3（注入上下文） | 同步 | 内联延迟 |
| `crisis.detected` | M5 | M3（中断生成）+ M5（推送前端） | **同步+广播** | < 100ms |
| `intent.changed` | M5 | M3（重载上下文）+ M4（切换 Layer3） | 同步 | < 50ms |
| `gate.insufficient` | M5 | M3（阻断生成）+ M5（推送追问 UI） | 同步 | < 200ms |
| `generation.completed` | M3 | M7（记录反馈埋点）+ M4（摘要触发） | 异步 | < 5s |
| `session.completed` | M4 | M4（压缩任务）+ M1（标签更新） | 异步 | < 30s |
| `profile.updated` | M1 | M6（缓存失效）+ M3（预热画像） | 异步 | < 10s |
| `perfumer_queue.updated` | M3 | M5（推送通知） | 异步 | < 60s |
| `error.degraded` | 任一模块 | M8（监控告警） | 异步 | < 5s |
| `graph.edge_updated` | M6 | M3（缓存失效） | 同步 | < 1s |

#### 3.2.3 事件结构规范

```
{
  "event_id": "evt_abc123",
  "event_type": "crisis.detected",
  "trace_id": "trace_xyz",        // 关联到请求入口
  "session_id": "sess_456",
  "timestamp": "ISO8601",
  "publisher": "M5",
  "payload": { ... },              // 事件特定数据
  "severity": "high" | "medium" | "low"
}
```

### 3.4 并发与一致性策略

| 场景 | 策略 | 细节 |
|------|------|------|
| **多标签页同用户** | 独立 session_id，Layer 2/3 共享 | 详见 FR-4.14。每个标签页独立 Layer 1 窗口。Layer 3 读取共享、写入乐观锁 |
| **压缩任务写摘要** | 外部存储先写 → Redis 后写 | 详见 FR-6.8。压缩任务独立于 session 生命周期。session 已关闭仍可写入 |
| **用户同时精炼 + 发新消息** | FIFO 排队 + 合并检测 | 详见 FR-4.12。两条精炼指令不同维度/同方向 → 合并为 1 次。反方向 → 分别执行 |
| **画像异步更新 vs 实时读取** | 读取最新版本号，不阻塞 | M3 读取时用快照版本。M1 异步更新不影响当前生成。下次请求取新版本 |
| **冷归档查询与热检索并发** | 热检索先返回，冷归档异步追加 | M4 同时发起两条查询。热检索 < 20ms 即刻返回。冷归档下一轮注入 |
| **SSE 推送与前端状态** | generation_id + phase 协议 | 详见 FR-4.13。前端 localStorage 绑定 generation_id → phase，断开后从断点续 |
| **Layer 1 溢出压缩** | 原子操作，压缩期间 Layer 1 只读 | 压缩任务获取写锁。若期间新消息到达 → 先排队，压缩完成后写入 |

#### 4.4.2 关键状态流转

**消息生命周期**:

```
用户输入
  → input.text / input.selected_cards
  → 发送按钮 disabled，input 清空
  → messages.append({role:"user", status:"sending"})
  → SSE chat.ack 到达
  → user_message.status → "sent"
  → (可选) chat.emotion 到达 → emotion state 更新 → EmotionConfirmation 渲染
  → (可选) gate.ask 到达 → gate state 更新 → GateQuestionBanner 渲染
  → gen.start 到达 → generation.id 设置，loading.phase = "skeleton"
  → gen.skeleton 到达 → messages.append(agent_msg with cards[] placeholder)
  → gen.detail 到达 → 逐卡片填充细节
  → gen.copy 到达 → 逐 chunk 追加文案
  → gen.complete 到达 → loading = null，generation 状态标记完成
  → 用户可: 精炼 / 收藏 / 分享 / 换风格
```

**精炼状态**:

```
用户输入精炼指令 "太甜了"
  → messages.append({role:"user", text:"太甜了"})
  → refine.start 到达 → loading.phase = "refining"
  → refine.result 到达 → 增量更新已有 cards[]
  → loading = null
  (如第 2 次精炼)
  → refine.gate 到达
    ├── continue → 同上
    └── downgrade_to_ask → refine.fallback → 展示追问文案
  (如第 3 次精炼失败)
  → refine.fallback {action:"upgrade_to_deep"} → gen.skeleton 返回全新推荐
```

**危机中断**:

```
safety.crisis 到达（在任意状态）
  → generation 状态标记 paused
  → SSE 中断
  → messages 中移除未完成的 agent 消息
  → messages.append({role:"agent", crisis_mode:true, text:"..."})
  → CrisisOverlay 组件渲染（展示热线 + 关怀文案）
  → input 区禁用推荐触发（文字仍可输入，但不触发生成）
  → session.status → "crisis_paused"
  → 用户新 session 后恢复
```

**降级可见性**:

```
gen.error / gen.complete 中 degraded=true
  → 对应卡片渲染时附淡色降级标注
  → 「基于已有信息推荐」文案（不阻断交互）
  → 埋点上报 degraded 事件
```

```
  quota:daily:{user_id}:sessions    → INCR, TTL=次日00:00
  quota:daily:{user_id}:generations → INCR, TTL=次日00:00
  quota:daily:{user_id}:deep_modes  → INCR, TTL=次日00:00
  quota:monthly:{user_id}:cards     → INCR, TTL=下月1日00:00
  quota:guest:{browser_id}:used     → SET, TTL=30天
```

### 5.3 计费点在全链路中的位置

```
用户消息到达
    │
    ├── [检查点 0] 游客? → guest_used 已存在? → 🔴 "你的免费体验已用完，注册后可继续 🌿"
    │
    ├── 输入处理（不计数）
    │
    ├── 新会话创建?
    │   └── [检查点 1] session_count < daily_limit?
    │       ├── ✅ → 创建，INCR
    │       └── 🔴 → "你今天的对话次数已用完，明天再来 🌿 还剩 X 小时后重置"
    │               前端展示倒计时（基于 Redis TTL）
    │
    ├── Gate 通过 → 即将进入生成
    │   └── [检查点 2] gen_count < daily_limit?
    │       ├── ✅ → 继续
    │       └── 🔴 → "你今天的推荐次数已用完 🌿"
    │
    ├── 即将升级深度模式
    │   └── [检查点 3] deep_count < daily_limit?
    │       ├── ✅ → 继续，INCR
    │       └── 🔴 → 降级为快速模式 + 提示
    │              "深度探索本日已用完，为你使用快速推荐 🌿"
    │
    └── 生成完成 → [计数] gen_count INCR（异步）
```

### 5.4 配额耗尽 UX

| 场景 | 触发时机 | 用户看到 | 用户可操作 |
|------|---------|---------|-----------|
| 游客用完 | 尝试第 2 次生成 | 「你的免费体验已结束 🌿」 | 注册 / 回首页 |
| 免费用户日会话耗尽 | 尝试创建新会话 | 「今天的对话次数已用完，X 小时后重置」+ 倒计时 | 浏览历史 / 升级 |
| 免费用户日生成耗尽 | 发送推荐请求 | 「今天的推荐次数已用完 🌿」 | 浏览历史 / 精炼已有推荐 / 升级 |
| 免费用户深度耗尽 | 触发深度条件 | 「深度探索本日已用完，为你使用快速推荐 🌿」 | 接受快速推荐 |
| 免费用户月卡片耗尽 | 点击制作卡片 | 「本月卡片制作次数已用完」 | 升级 / 下月再来 |
| 接近限额（80%） | 生成完成后 | 淡色提示「今日还可生成 X 次」 | 无操作，仅提示 |

```
│                                     │

└─────────────────────────────────────┘
```

### 5.6 技术约束

| 约束 | 方案 |
|------|------|
| 计数延迟 | Redis INCR 原子操作，< 1ms |
| 并发安全 | Redis 单线程保证 INCR 不超卖 |
| 计数回滚 | 生成失败不扣减（先 INCR 后生成 → 失败时 DECR） |
| 跨日重置 | Redis TTL = 次日 00:00 自动过期，无需定时任务 |
| 跨设备一致 | 以 user_id 为 key，多标签页共享同一计数 |
| 降级 | Redis 不可用 → 跳过配额检查（不断用户路），标记 `quota_degraded` |
| 审计 | PG 日志表记录每次 INCR/DECR（异步写入，不阻塞检查） |
