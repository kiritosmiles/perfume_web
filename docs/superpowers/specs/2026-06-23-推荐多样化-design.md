## 问题分析

### 现象

同一个情绪+场景组合下，推荐结果总是相同的 3 款香水（以 `joy` 为例：Spiced Citrus / Rose Bershka / Citrus Oud），与其他情绪、场景、文本的交互差异极小。

### 根因

```
用户输入 "开心" + "约会"
        │
        ▼
  resolve_emotion_from_cards
  → joy=0.9, excitement=0.7, calm=0.2, romance=0.1, ...
        │
        ▼
  fragrance.py: sorted_emotions[:2]                    ← 只用 top 2
  → joy=0.9, excitement=0.7
        │
        ▼
  SOOTHES 边: joy → citrus/fruity/floral (3条)        ← 每情绪 2-3 条
              excitement → sweet/aromatic (2条)
        │
        ▼
  1179 款香水在 5 个 Accord 池中竞争                    ← 池太小
        │
        ▼
  按 total_accord_score + rating*0.5 排序
  场景 scene_tag 不参与评分                             ← 场景无效
        │
        ▼
  build_skeleton: candidates[:3]                       ← 纯分数，无多样性
        │
        ▼
  始终同一 3 款 → 用户感知"总是这些"
```

### 根本原因总结

1. **情绪→香韵边太少**：8 情绪 × 2.75 条边 = 22 条，每个情绪覆盖的 Accord 不到 Accord 总数 (70) 的 5%
2. **只用 top 2 情绪**：低权重情绪（如 calm=0.2, romance=0.1）完全不影响推荐
3. **场景零参与**：`scene_tag` 传入了后端但在评分中不起作用
4. **无多样性约束**：3 个结果可能全属同一个香韵簇

## 设计方案

### 整体架构

```
用户输入 (cards + text + scene)
        │
        ▼
  emotion_vector (8维, resolve不变)
        │
        ▼
  fragrance.py ──── 全部 8 情绪 × ~9.4 条边/情绪 = ~75 条路径
    │                  + scene_tag 匹配加分
    │                  + limit=50 (更大候选池)
        ▼
  ~50 candidates (按 score 排序)
        │
        ▼
  generation.py ─── _diverse_top3() 香韵聚类贪心
    │
        ▼
  3 款推荐 (不同香韵簇)
```

### 改动 ①：情绪→香韵映射扩容

从每个情绪 2-3 条边扩展到 8-10 条，分三层权重：

```
Tier 1 (0.90-0.95): 最核心香韵 — 主导推荐方向
Tier 2 (0.70-0.85): 辅助香韵 — 增加多样性
Tier 3 (0.45-0.65): 弱关联香韵 — 拉远距匹配（如 joy→green）
```

完整映射表：

| 情绪 | Tier 1 | Tier 2 | Tier 3 |
|------|--------|--------|--------|
| **joy** 开心 | citrus 0.94 | fruity 0.85, floral 0.82, sweet 0.75 | fresh 0.65, aromatic 0.60, tropical 0.55, green 0.50, white_floral 0.45 |
| **calm** 平静 | woody 0.93 | musky 0.82, powdery 0.78, fresh 0.70 | green 0.65, aquatic 0.58, herbal 0.55, lavender 0.50, amber 0.45 |
| **sadness** 难过 | amber 0.90 | warm_spicy 0.82, woody 0.78, vanilla 0.72 | powdery 0.62, musky 0.58, floral 0.52, soft_spicy 0.48, balsamic 0.45 |
| **anxiety** 焦虑 | vanilla 0.92 | citrus 0.83, lavender 0.80, fresh 0.72 | powdery 0.62, green 0.58, musky 0.52, sweet 0.48, aquatic 0.45 |
| **excitement** 兴奋 | sweet 0.88 | aromatic 0.82, citrus 0.78, fruity 0.72 | warm_spicy 0.65, amber 0.60, vanilla 0.55, woody 0.50, tropical 0.48 |
| **nostalgia** 怀旧 | woody 0.92 | leather 0.82, earthy 0.78, amber 0.72 | warm_spicy 0.65, oud 0.60, patchouli 0.55, tobacco 0.50, smoky 0.48 |
| **romance** 浪漫 | floral 0.94 | rose 0.85, powdery 0.82, sweet 0.75 | vanilla 0.65, fruity 0.60, musky 0.55, amber 0.50, white_floral 0.48 |

**总计：75 条 SOOTHES 边**（从 22 条扩展）

### 改动 ②：全部 8 情绪参与


### 改动 ③：场景参与评分

```cypher
// 新增 scene boost
OPTIONAL MATCH (p)-[ss:SUITS_SEASON]->(:Scene {name: $scene_tag})
...
WITH p, b, ss, total_accord_score, matched_accords,
     total_accord_score
     + COALESCE(toFloat(p.rating), 3.0) * 0.5          -- rating tiebreaker (max 2.5)
     + CASE WHEN ss IS NOT NULL THEN 0.25 ELSE 0 END   -- scene boost
     AS score
```

scene boost = 0.25，相当于 0.5 分 rating 加成——足够区分但不主导。

### 改动 ④：香韵聚类 + 贪心多样性

**香韵聚类**：将 70 个 Accord 归入 9 个大类，按嗅觉相似性：

| 簇 ID | 簇名 | 包含 Accord |
|--------|------|------------|
| citrus | 柑橘系 | citrus, fruity, tropical, fresh |
| floral | 花香系 | floral, rose, white floral, yellow floral, violet, iris, tuberose |
| woody | 木质系 | woody, earthy, mossy, patchouli, oud, conifer, green |
| spicy | 辛香系 | amber, warm spicy, fresh spicy, soft spicy, aromatic, cinnamon, herbal, anis |
| sweet | 甜香系 | sweet, vanilla, caramel, honey, chocolate, cacao, almond, coconut, gourmand |
| musky | 麝香系 | musky, powdery, soapy, aldehydic, balsamic, lactonic, milky |
| leather | 皮革系 | leather, animalic, smoky, tobacco |
| aquatic | 海洋系 | aquatic, marine, ozonic, lavender, fresh spicy |
| other | 其他 | metallic, mineral, salty, alcohol, coffee, rum, camphor, beeswax, cherry, wine, whiskey, champagne, vodka, rubber, sand, sour, bitter, terpenic, nutty, savoury |

**贪心算法**：

```python
def _diverse_top3(candidates: list[dict]) -> list[dict]:
    selected = []
    used_clusters = set()
    for c in sorted(candidates, key=lambda c: c.get("score", 0), reverse=True):
        accord = c.get("accord", "unknown")
        cluster = ACCORD_CLUSTERS.get(accord, "other")
        if cluster not in used_clusters:
            selected.append(c)
            used_clusters.add(cluster)
        if len(selected) >= 3:
            break
    # Fallback: if < 3 after diversity (unlikely with 9 clusters), fill with best remaining
    if len(selected) < 3:
        for c in sorted(candidates, key=lambda c: c.get("score", 0), reverse=True):
            if c not in selected:
                selected.append(c)
            if len(selected) >= 3:
                break
    return selected
```

**效果示例**（joy=0.9, scene=date）：

```
旧: Spiced Citrus(citrus) + Rose Bershka(floral) + Citrus Oud(citrus)
    → 2/3 同属 citrus 簇 ❌

新: Spiced Citrus(citrus) + Rose Bershka(floral) + Bleu de Chanel(woody)
    → 3 个不同簇 ✅
```
