## §1 信息架构

> 详见业务流设计文档 §4.1.1

### 1.1 页面路由树

```
/                          → 应用主页（介绍页 / Landing Page）
├── /chat                   → 对话主页面（核心）
│   ├── /chat?intent=self   → 自用模式
│   ├── /chat?intent=gift   → 送礼模式
│   └── /chat?scene=spring  → 场景快捷入口
├── /guest                  → 游客体验页（免注册，单次对话）
├── /onboarding             → 新用户引导（≤ 3 题，详见 FR-1.2）
├── /history                → 历史会话列表（按天聚合，详见 FR-4.11）
│   └── /history/{sessionId} → 历史会话详情（只读）
├── /profile                → 「AI 眼中的我」（详见 FR-4.8）
│   ├── /profile/personality → 人格画像展示
│   ├── /profile/memory      → 记忆管理（查看/删除摘要）
│   ├── /profile/safety      → 安全档案编辑（过敏原/不喜欢）
│   └── /profile/settings    → 通知偏好 / 数据删除（详见 FR-9.4）
├── /my-cards               → 我的配方卡片（调香师队列状态，详见 FR-3.10）
├── /mood-journal           → 情绪日记（详见 FR-4.9 回访入口）
│   └── /mood-journal/{weekId} → 某周情绪周记
├── /share/{shareId}        → 分享落地页（只读卡片预览）
├── /crisis-resources       → 心理援助资源静态页（详见 FR-5.6 可配置）
├── /privacy                → 隐私说明
└── /fallback               → 服务降级静态页
```

**路由参数说明**:

| 路由 | 查询参数 | 用途 |
|------|---------|------|
| `/chat?intent=self` | intent=self | 直接进入自用模式对话 |
| `/chat?intent=gift` | intent=gift | 直接进入送礼模式对话 |
| `/chat?scene=spring` | scene=spring | 季节场景快捷入口 |
| `/history/{sessionId}` | sessionId | 查看特定会话的完整对话记录（只读） |
| `/mood-journal/{weekId}` | weekId | 查看特定周的情绪周记 |

## §2 应用主页

> 详见业务流设计文档 §4.1.0

### 2.1 页面定位

应用主页是用户进入网站的第一个触点，承担三个职责：

| 职责 | 说明 |
|------|------|
| **价值传达** | 一句话讲清楚"这个 AI 能为你做什么"，降低跳出率 |
| **降低门槛** | 让用户无需注册即可快速体验（游客入口），体现 C 端设计原则"低门槛" |
| **分流导航** | 根据用户状态分流：新访客 → 游客体验 / 已有账号 → 登录 / 回归用户 → 继续对话 |

### 2.2 页面结构 (LandingPage)

```
LandingPage
├── HeroSection                       // 首屏
│   ├── Tagline                       // "用香氛，读懂你的情绪 🌿"
│   ├── Subtitle                      // "AI 驱动的情绪×香水推荐——你说说心情，它为你找到专属气味"
│   ├── CTA Button (Primary)          // 「✨ 免费体验」→ /guest
│   ├── CTA Button (Secondary)         // 「登录 / 注册」
│   └── Visual: 情绪卡片旋转展示动画    // 8 张情绪卡片缓缓轮播
│
├── HowItWorksSection                 // 三步说明
│   ├── Step 1: 「说出你的心情」
│   │   └── 文字或选一张情绪卡片，AI 听懂你的感受
│   ├── Step 2: 「AI 为你解读」
│   │   └── 结合你的个性与偏好，从上千种香调中为你搭配
│   └── Step 3: 「发现你的专属气味」
│       └── 获得香氛配方 + 文字解读，还可找调香师为你调制实体卡片
│
├── FeaturesSection                   // 核心能力展示
│   ├── FeatureCard: 「情绪感知」
│   │   └── 不只是文字——AI 理解你话语背后的情绪层次
│   ├── FeatureCard: 「场景智能」
│   │   └── 为自己选 / 为重要的人挑礼物，AI 自动切换推荐策略
│   ├── FeatureCard: 「被记住」
│   │   └── 每次对话都被用心记住，越来越懂你
│   └── FeatureCard: 「专业调香师背书」
│       └── Helen Yee 团队为你的专属配方调制实体香氛卡片
│
├── TestimonialSection (可选)           // 用户反馈 / 示例对话摘录
│   └── 示例对话片段：展示 AI 的关怀语气和推荐风格
│
├── GuestEntrySection                 // 游客快捷入口（二屏强化）
│   ├── 「不需要注册，现在就开始 🌿」
│   ├── 情绪卡片快捷选择（8 选 1~2）
│   ├── 场景标签快捷选择
│   └── 「开始探索」→ POST /guest/sessions → 进入 /guest
│
└── Footer
    ├── 链接: 隐私说明 / 心理援助资源 / 关于我们
    └── 版权信息
```

### 2.3 分流逻辑

```
用户访问 /
    │
    ├── 已登录（有有效 token）
    │   ├── 有活跃 session → 直接进入 /chat
    │   ├── 无活跃 session（> 30min 未活动）→ 展示主页 + 「继续对话 🌿」CTA
    │   └── 首次登录（新用户）→ 展示主页 + 「✨ 开始你的第一次探索」CTA
    │
    ├── 游客（有临时 token，游客模式进行中）
    │   └── 展示主页 + 「继续刚才的对话 →」→ /guest
    │
    └── 新访客（无 token）
        └── 完整主页展示 → 引导「✨ 免费体验」
```

### 2.4 路由行为表

| 用户状态 | 访问 `/` | 行为 |
|---------|---------|------|
| 新访客 | 首次 | 完整展示主页，不含"继续对话"入口 |
| 游客进行中 | 再次 | Hero CTA 变为「继续刚才的探索 →」 |
| 已登录，活跃 session | — | 自动重定向到 `/chat`（不展示主页） |
| 已登录，无活跃 session | — | Hero CTA 变为「继续对话 🌿」→ 新 session |
| 已登录，新注册 | 首次 | Hero CTA 变为「✨ 开始你的第一次探索」→ /onboarding |

### 2.5 情绪卡片快捷入口

主页 GuestEntrySection 中嵌入简化版情绪输入组件：

```
卡片列表（单选，最多可选 2 张）:
  😊 开心   😢 难过   😰 焦虑   😌 平静
  🎉 兴奋   🥺 怀旧   💕 浪漫   🌧️ 忧郁

场景标签（可选，单选）:
  💼 通勤工作   💑 约会之夜   🏠 宅家放松
  🎊 聚会社交   🎁 挑选礼物   🔍 随便看看

点击「开始探索」→ 所选内容作为第一次对话的初始输入
```

## §4 关键页面线框

### 4.1 冷启动引导 (FR-1.2)

> 详见需求文档 FR-1.2

#### 概述

极简场景化引导问卷，新用户注册后首访 `/onboarding`。用场景替代术语，降低认知门槛。

#### 设计要求

| 维度 | 规格 |
|------|------|
| 题目数量 | ≤ 3 题 |
| 语言风格 | 无香调术语，使用日常场景化描述 |
| 交互模式 | 一屏一题，逐步展开 |
| 响应速度 | 每个选项背后预设映射向量，无需 LLM，毫秒级响应 |
| 退出机制 | 用户可跳过任意题或全部跳过（标记冷启动画像稀疏） |

#### 页面布局

```
OnboardingPage
├── ProgressIndicator          // 进度点 (● ○ ○)
├── QuestionCard               // 当前问题
│   ├── QuestionText           // 场景化提问文案
│   └── OptionList             // 2-4 个选项（大卡片/按钮）
├── SkipButton                 // 「跳过」文字链接
└── VisualBackground           // 柔和的渐变背景 + 香气意象
```

#### 示例问题结构

| 题序 | 场景提问示例 | 选项 | 映射维度 |
|:---:|------------|------|---------|
| Q1 | "你通常在什么场合用香水？" | 日常通勤 / 社交聚会 / 独处放松 / 特别约会 | 场景偏好向量 |
| Q2 | "你更希望香水给你带来什么感觉？" | 自信大方 / 温柔亲近 / 神秘独特 / 清新活力 | 人格-香调映射 |
| Q3 | "你喜欢什么样的气味氛围？" | 自然花草 / 温暖木质 / 清新柑橘 / 甜美果香 | 香调偏好基线 |

## §5 SSE 交互时序

> 详见业务流设计文档 §4.2.2 和 §4.2.3

### 5.1 SSE 事件域命名规范

```
event: {domain}.{phase}[:{subtype}]

domain:  chat     → 对话消息
         gen      → 配方生成
         refine   → 精炼
         gate     → Agent Gate 交互
         safety   → 安全审核
         lifecycle → 生命周期
         system   → 系统级通知
```

### 5.2 完整事件目录（前端视角）

以下从**前端 UI 行为**角度列出所有 SSE 事件及其对应的 UI 变化。

#### 对话消息域 (`chat.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `chat.ack` | 服务端确认收到消息 | 用户消息气泡状态: `sending` → `sent`；显示双勾确认 |
| `chat.emotion` | 情绪识别完成 | 渲染 EmotionConfirmation 组件: 展示 `primary_emotion` + 情绪标签 + 置信度。「不对 ✏️」按钮可点击 |
| `chat.intent` | 意图检测完成 | IntentToggle 高亮当前意图图标；送礼模式时切换输入引导文案 |
| `chat.error` | 消息处理失败 | 展示错误提示 + 重试按钮（如果 `retryable=true`） |

#### 生成域 (`gen.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `gen.start` | 生成开始 | 创建 AgentMessage 占位气泡；loading 态设置为 "skeleton"；展示 ThinkingIndicator |
| `gen.skeleton` | ⭐ 骨架阶段完成（首屏可见） | **关键 UI 节点**: FragranceCardSkeleton 填充为真实卡片骨架: 香调组合 + 匹配分数出现。用户看到第一批可交互内容 |
| `gen.detail` | 逐卡片补充详情 | 对应 rank 的卡片展开更多字段: 完整匹配维度、推理路径等 |
| `gen.copy` | 解读文案流式 | 逐 chunk 追加 StorytellingCopy 文本（打字机效果）；`is_final=false` 时持续追加，`is_final=true` 时完成 |
| `gen.complete` | 生成完成 | ThinkingIndicator 消失；ActionBar (收藏/分享/换风格) 可交互；记录 `generation_id` 完成 |
| `gen.error` | 生成失败 | 展示错误文案；如 `degraded=true` → 展示 DegradedNotice 淡色提示 |

#### Agent Gate 域 (`gate.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `gate.check` | Gate 判定结果返回 | `verdict=sufficient` → 无 UI 变化（0ms 开销）；`verdict=partial` → 展示 ThinkingIndicator "正在查找相关信息..." |
| `gate.ask` | 信息不足，输出追问 | ⭐ **首响应可见**: GateQuestionBanner 渲染，展示 ≤3 个追问 + 跳过按钮「先推荐看看」 |
| `gate.wait` | 工具补充进行中 | 短暂展示 "正在查找相关信息..." 加载态 |

#### 精炼域 (`refine.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `refine.start` | 精炼开始 | loading 态设置为 "refining"；精炼指令消息气泡状态更新 |
| `refine.result` | 精炼结果返回 | 增量更新已有 FragranceCard[]: 调整的维度高亮闪烁 |
| `refine.gate` | 第 2 次精炼语义判断结果 | `verdict=continue` → 同 refine.result；`verdict=downgrade_to_ask` → 展示追问文案 |
| `refine.fallback` | 精炼失败 | `action=ask` → 展示追问文案；`action=upgrade_to_deep` → 文案「让我换个思路...」+ 触发全新深度生成 |

#### 安全域 (`safety.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `safety.ok` | 安全审核通过 | 无 UI 变化（不推送） |
| `safety.warn` | 低/中风险提醒 | 淡色提示条 "部分内容已根据安全策略调整" |
| `safety.crisis` | 🔴 危机检测 | **SSE 中断**；CrisisOverlay 渲染: 关怀文案 + 热线列表；input 区禁用推荐触发；session 标记 `crisis_paused` |
| `safety.block` | 内容被拦截 | 用户消息显示为 "消息已发送 ✅"（不展示原文）；不生成回复 |

#### 生命周期域 (`lifecycle.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `lifecycle.session` | 会话状态变更 | `idle_timeout` → 展示 "会话已结束 🌿" 提示；`completed` → 更新 session 状态栏 |
| `lifecycle.resume` | 断点续传就绪 | 从 `from_phase` 恢复 SSE 流，前端准备接收对应阶段事件 |

#### 系统域 (`system.*`)

| 事件 | 触发时机 | UI 变化 |
|------|---------|--------|
| `system.heartbeat` | 每 15s | 内部时间戳更新（无 UI 变化，用于连接保活） |
| `system.notification` | 异步通知 | `perfumer_update` → 右上角红点提示；`mood_journal_ready` → 跳转入口高亮 |
| `system.error` | 系统级错误 | 全局错误提示条；如可重试则展示重试按钮 |

### 5.3 前端 SSE 客户端状态机

```
           ┌──────────┐
           │  IDLE     │ ← 未连接
           └─────┬────┘
                 │ connect(url, session_id)
                 ▼
           ┌──────────┐
           │CONNECTING│
           └─────┬────┘
                 │
        ┌────────┼────────┐
        ▼        ▼        ▼
   ┌────────┐ ┌──────┐ ┌──────────┐
   │ ACTIVE │ │ERROR │ │UNAUTHORIZED│
   │        │ │(retry│ │(redirect  │
   │        │ │able) │ │ to login) │
   └───┬────┘ └──┬───┘ └──────────┘
       │         │
       │    ┌────┴────┐
       │    │ retry?  │
       │    ├─────────┤
       │    │yes: 指数│
       │    │退避 1s→ │
       │    │2s→4s(max│
       │    │5次)     │
       │    └────┬────┘
       │         │ no
       │         ▼
       │    ┌──────────┐
       │    │DISCONNECTED│ → 展示"连接中断"UI + 手动重试按钮
       │    └──────────┘
       │
       │ 中断恢复逻辑:
       │ 重连成功 → 检查 localStorage(generation_id, last_phase)
       │ ├── 有未完成生成 → POST /generation/{id}/resume
       │ └── 无 → 加载最新消息
       │
       └── EVENT 到达 → 分发到对应状态处理器 → 更新 UI

指数退避: [1s, 2s, 4s, 4s, 4s]（最多 5 次，总等待 ≤ 15s）
5 次失败后 → DISCONNECTED，展示手动重试按钮
```

#### 事件分发到 UI 更新

```
EVENT 到达
  → SSE Client 解析 event type
  → 分发到 domain handler:
      chat.*    → ChatStateHandler → 更新 messages[] / emotion / intent 状态
      gen.*     → GenerationStateHandler → 更新 generation / cards[] / loading
      gate.*    → GateStateHandler → 更新 gate / messages[] (追加追问气泡)
      refine.*  → RefineStateHandler → 增量更新 cards[] / loading
      safety.*  → SafetyStateHandler → 更新 crisis flag / 切换 UI 模式
      lifecycle.* → SessionStateHandler → 更新 session.status
      system.*  → NotificationHandler → 更新通知 badge / error banner
  → React/Vue 响应式状态 → 组件自动重渲染
```

### 6.1 状态分层架构

├─────────────────────────────────────────┤
│ Session State (per-session store)        │
│ ├── session: {id, status, intent}        │
│ ├── emotion: {vector, confidence, source}│
│ ├── messages: Message[]                  │
│ ├── generation: {id, phase, cards[]}     │
│ └── gate: {verdict, questions[]}         │
├─────────────────────────────────────────┤
│ UI State (component-local)               │

### 6.2 关键状态流转

#### 6.2.1 消息生命周期

```
用户输入
  → input.text / input.selected_cards
  → 发送按钮 disabled，input 清空
  → messages.append({role:"user", status:"sending"})
  → SSE chat.ack 到达
  → user_message.status → "sent"
  → (可选) chat.emotion 到达 → emotion state 更新 → EmotionConfirmation 渲染
  → (可选) gate.ask 到达 → gate state 更新 → GateQuestionBanner 渲染
  → gen.start 到达 → generation.id 设置，loading.phase = "skeleton"
  → gen.skeleton 到达 → messages.append(agent_msg with cards[] placeholder)
  → gen.detail 到达 → 逐卡片填充细节
  → gen.copy 到达 → 逐 chunk 追加文案
  → gen.complete 到达 → loading = null，generation 状态标记完成
  → 用户可: 精炼 / 收藏 / 分享 / 换风格
```

#### 6.2.2 精炼状态

```
用户输入精炼指令 "太甜了"
  → messages.append({role:"user", text:"太甜了"})
  → refine.start 到达 → loading.phase = "refining"
  → refine.result 到达 → 增量更新已有 cards[]
  → loading = null
  (如第 2 次精炼)
  → refine.gate 到达
    ├── continue → 同上
    └── downgrade_to_ask → refine.fallback → 展示追问文案
  (如第 3 次精炼失败)
  → refine.fallback {action:"upgrade_to_deep"} → gen.skeleton 返回全新推荐
```

#### 6.2.3 危机中断

```
safety.crisis 到达（在任意状态）
  → generation 状态标记 paused
  → SSE 中断
  → messages 中移除未完成的 agent 消息
  → messages.append({role:"agent", crisis_mode:true, text:"..."})
  → CrisisOverlay 组件渲染（展示热线 + 关怀文案）
  → input 区禁用推荐触发（文字仍可输入，但不触发生成）
  → session.status → "crisis_paused"
  → 用户新 session 后恢复
```

#### 6.2.4 降级可见性

```
gen.error / gen.complete 中 degraded=true
  → 对应卡片渲染时附淡色降级标注
  → 「基于已有信息推荐」文案（不阻断交互）
  → 埋点上报 degraded 事件
```

### 7.4 精炼对话 UI (FR-3.7)

**定位**: 用户在推荐结果上提调整需求时的交互组件。

```
精炼输入:
  TextArea placeholder: "太甜了？太浓了？告诉我你的感受..."
  或快捷按钮:
    「更甜一点」「更清淡」「更清新」「更温暖」「换个香调」

精炼响应 (RefinementResponse):
├── 规则可处理 (< 500ms):
│   ├── 调整确认: "好的，为你调整了甜度——"
│   └── 增量更新的 FragranceCard (调整维度闪烁高亮)
├── Agent Gate 语义不满 (第 2 次精炼):
│   ├── 展示追问文案:
│   │   "让我换个思路——
│   │    你刚才说的[情绪/场景]，我理解对了吗？
│   │    可以告诉我更多关于[缺失维度]吗？"
│   └── 展示可选快捷回复
└── 第 3 次精炼仍失败:
    ├── 文案: "你已经调整了 3 次——让我换个思路，从不同的角度重新为你搭配 🌿"
    └── 触发深度模式，generation.skeleton 返回全新推荐
```

**精炼规则维度 (9 个)** -- 穷举覆盖:

| 维度 | 可处理指令示例 |
|------|-------------|
| 甜度 | "太甜"/"不够甜"/"更甜一点" |
| 浓度 | "太浓"/"太淡"/"更清淡" |
| 清新度 | "更清新"/"太沉闷" |
| 性别倾向 | "更女性化"/"更男性化"/"太中性" |
| 温暖度 | "太冷"/"更温暖" |
| 香调切换 | "换个香调"/"试试花香" |
| 场合切换 | "更正式"/"更休闲" |
| 年龄适配 | "太年轻"/"太成熟" |
| 季节适配 | "更适合夏天用" |

**不可处理**: "太贵了"(价格,MVP范围外) / "前调太冲"(三调精细调整) → 走 Agent Gate 追问或升级。

**定位**: 推荐结果上的多样性控制。

```
多样化控件:
├── DiversifyButton: 「🔀 换个风格」

│   │   ├── 目标风格余弦距离 < 0.6 → 快速模式（GraphRAG 近邻变体）
│   │   └── 目标风格余弦距离 ≥ 0.6 → 触发深度模式（完全不同的香调类别）

└── DiversitySlider: (在设置或推荐结果中可选)

    └── 激进: 更多意外和惊喜
```

### 7.6 意图切换 (FR-5.7)

**定位**: ChatHeader 中的三态切换控件。

```
IntentToggle
├── 三态切换:
│   ├── 🌿 为自己 (self_use)    // 主动态
│   ├── 🎁 为他人 (gift)         // 中态
│   └── 🔍 随便看看 (explore)    // 末态
├── 交互: 点击切换，即时生效
├── 手动切换直接覆盖 LLM 判定
├── 视觉: 当前选中态高亮 + 底色
└── 切换动画: 平滑过渡 200ms

意图变化时的确认微交互:
  如在精炼中检测到意图变化 (self_use → gift):
  → 轻量弹窗: "你是想为自己调整还是为她调整？🌿 为自己 / 🎁 为她"
  → 不等 Agent Gate 的二次精炼
```

### 7.7 Agent Gate 追问 (FR-5.11)

**定位**: 推理前信息不足时，在对话流中展示追问横幅。

```
GateQuestionBanner
├── 温和引导文案: "先了解几个小细节 🌿"
├── QuestionList (≤ 3 个问题):
│   ├── ① "收礼人平时工作是什么类型？"
│   ├── ② "之前有没有用过她很喜欢的香水？"
│   └── ③ "她的皮肤对某些气味敏感吗？"
├── 交互区域:
│   ├── 每道题旁有快捷回复按钮 (可选)
│   ├── TextArea: 用户可自由回答
│   └── SkipButton: 「先推荐看看」→ 标记 partial 模式，继续生成
└── 设计约束:
    ├── 不涉及 PII（正则过滤）
    ├── 对话式引导，非表格化语气
    └── 追问展示前经本地敏感词过滤
```

### 7.8 危机响应 (FR-5.6)

**定位**: 危机检测后全局切换的 UI 模式。

```
CrisisOverlay (覆盖对话区或全屏)
├── 关怀文案 (禁止任何推荐语气):
│   "我注意到你提到了一些让我担心的内容。
│    作为 AI，我无法提供专业心理支持，
│    但希望你知道——你不需要一个人面对这些。"
│
├── HotlineSection:
│   ├── HotlineCard[]
│   │   ├── 机构名称
│   │   ├── 电话号码 (可点击拨打)
│   │   └── 服务区域
│   └── 热线数据来源: FR-5.6 的可配置 JSON，
│       按用户地区匹配展示
│
├── Session 限制提示:
│   "本轮对话中，我不会主动推荐香水。
│    但你可以继续和我聊天。"
│
└── 恢复机制:
    └── 用户手动切换意图标签 或 新 session → 恢复正常
```

**危机 vs 普通负面情绪区分**:

| 维度 | 普通负面情绪 | 危机信号 |
|------|-----------|---------|
| 关键词 | "焦虑""难过""压力大" | "不想活了""结束一切""没有希望" |
| 语境 | 有具体触发事件 | 无具体事件，或事件 + 自我否定 |
| 时间维度 | 临时状态 | 持续/永久 |
| 行为意向 | 寻求帮助/建议 | 表达无望/结束意图 |

### 7.9 降级提示

**定位**: LLM 降级后在各受影响内容区域的淡色标注。

```
DegradedNotice
├── 展示位置: 对应卡片 / 文案下方
├── 样式: 淡色小字，不阻断交互
├── 文案:
│   ├── 默认: 「基于已有信息推荐」
│   ├── safety_degraded: 「安全审核暂时降级」
│   ├── quality_degraded: 「推荐质量可能略有下降」
│   └── graph_fallback: 「基于 AI 香水知识推荐」
└── 触发条件: LLM 调用降级标记 `degraded=true`
```

| 层 | 选择 | 理由 |

| 样式 | Tailwind CSS | 快速迭代品牌调性 |
