## P0 — 关键缺失（阻塞核心流程）

### P0.1 SSE refine.* 事件处理

**文件范围**: packages/frontend/src/hooks/useSSE.ts

**什么是缺失的**: 3 个精炼相关 SSE 事件未被 useSSE.ts 消费:

| 事件 | 缺少的处理行为 |
|------|-----------|
| refine.start | 应在 generationStore 中设置 refineAttempt 和 refineMethod |
| refine.result | 应调用 generationStore 更新卡片数据（adjustments + updated_cards） |
| refine.gate | 应切换精炼策略：continue → 继续等待结果；downgrade_to_ask → 展示 GateQuestionBanner |
| refine.fallback | 应处理精炼回退：ask → 展示 Gate 追问；upgrade_to_deep → 更新 mode 状态 |

**现状影响**: RefinementChips 点击后前端不会感知精炼进度，用户看不到正在优化的中间状态，精炼完成后也无法收到结果更新。

**修改方案**:
1. generationStore 新增 refineAttempt, refineMethod, pendingRefinement 状态
2. useSSE.ts 新增 4 个 case 分支
3. RecommendationFlow 新增 refinePhase 驱动的 UI 反馈

**工作量**: ~40 行 store + ~30 行 useSSE + ~20 行 UI

---

### P0.2 SSE lifecycle.* 事件处理

**文件范围**: packages/frontend/src/hooks/useSSE.ts, packages/frontend/src/stores/sessionStore.ts

| 事件 | 缺少的处理行为 |
|------|-----------|
| lifecycle.session | 应更新 sessionStore 中 sessionStatus（active / idle_timeout / completed）|
| lifecycle.resume | 应设置断点续传状态，允许从 from_phase 恢复渲染 |

**修改方案**:
1. sessionStore 新增 sessionStatus 和 resumeInfo 字段
2. useSSE.ts 新增 2 个 case 分支
3. RecommendationFlow 新增 sessionStatus 驱动的 UI 提示

**工作量**: ~20 行 store + ~15 行 useSSE + ~30 行 UI

---

### P0.3 SSE 系统事件处理

**文件范围**: packages/frontend/src/hooks/useSSE.ts

| 事件 | 缺少的处理行为 |
|------|-----------|
| system.heartbeat | 应重置心跳超时计时器 |
| system.notification | 应触发 UI 提示（perfumer_update / mood_journal_ready）|
| system.error | 应设置 UI 错误状态 |

**修改方案**:
1. sessionStore 新增 notifications 队列和 systemError 字段
2. useSSE.ts 新增 3 个 case 分支
3. 新增 SystemNotificationToast 组件
4. App.tsx 添加全局 systemError Banner

**工作量**: ~25 行 store + ~20 行 useSSE + ~50 行组件
